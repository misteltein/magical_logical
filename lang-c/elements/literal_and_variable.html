リテラルと変数

リテラルとは，直値とも言われ，ソースコードに直に値を書いたものである。 変数は，値を保存しておくためのもので，変更することができる。 これら値はバイナリで表されており，これを計算機がどう解釈するか決めるために型情報が必要になる。 この型は，変数を作成する際に指定する。

値の出力については別のページで説明する。

文字列を表すリテラルは，ダブルクオートで挟み表記する。 これは char 型の配列に保存することができる。

整数

クオーテーションを使わず直に整数を書いたものが整数リテラルとなる。 整数を表すデータ型は int 型である（他にもある）。

負の整数は，整数を表すリテラルに単項演算子 - をつけて表すことができる。

浮動小数点数

浮動小数点数のリテラルもクオーテーションを使わずに書く。 浮動小数点数を表すデータ型には float 型と double 型がある。

float 型は 4 byte, double 型は 8 byte を使って数を表す。 符号，指数部，仮数部それぞれを表すために何 bit を使うかは処理系によって変わるため， 精度は環境に依存する。

整数の場合と同じく，リテラルに単項演算子を付けて符号を指示することができる。

型修飾子 long を指定することで double 型より表せる数の範囲が広がる（処理系依存）。

接尾辞を付けない場合には double, F を付けた場合には float， L を付けた場合には long double としてそれぞれ解釈される。

指数表記を使ってリテラルを与えることもできる。 仮数部と指数部を E で繋げたものがリテラルとなる。

定数

型の指定のまえに const 修飾子をつければ，この変数は値の変更が許されない定数となる。 つまり変数を特殊化することにより定数を用意している。

のように書けば PI は定数となる。 定数の名前は大文字で書く慣習があるが，プログラム全体で統一されていれば問題ない。

このように定数としておけば，リテラルを散りばめることがなくなり， この宣言１文を書き換えるだけでプログラム全体の振る舞いを変えることができるようになるため， 活用してほしい。
