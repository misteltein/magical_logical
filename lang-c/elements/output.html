標準出力

ターミナルなどのコンソールへの標準出力は，プログラム処理の結果を表示するだけでなく， その途中が正しいか確かめるときにも必要である。

printf 関数を使った標準出力

printf 関数は，リテラルや，変数の値を標準出力するために使う。

表示したいものがリテラルだけである場合には，それを引数に与える。

変数の値を含めたり，または変数の値のみを出力することもできる。

このダブルクオーテーションで括られた文字列はプレースホルダ（仮置きの文字列）である %d を含んでいる。 これを height という変数の値で置き換えて， 完成した 「height: 141 cm」 という１文が標準出力される。

類似した標準出力用の関数は他にもあるので，調べてみてほしい。

フォーマット指定子

先程紹介した %d のようなプレースホルダは，単なる仮置きの文字列ではない。 このプレースホルダはフォーマット指定子と呼ばれるものであり， 表示する際に必要なデータ型を指示するとともに， その出力のフォーマット（形式）を指定するための役割を持っている。

データ型を指定する指定子は必須である。 そして以下のように指定子と型が対応している。

指定子対応する型出力するもの
%dint符号付き整数（10 進数）
%uunsigned int符号なし整数（10 進数）
%ffloat, double浮動小数点数
%cchar1 文字
%schar[], char*文字列

プログラムの処理を確かめる際に値の出力を頻繁に行うため，これらはとても重要である。

詳細なフォーマットの指定

先程の表に示したフォーマット指定子は、変換指定子と呼ばれるものでデータの型のみを指定していたが、 より詳細な指定も可能である。

フォーマット指定子の構文

出力フォーマットは，フラグ (flags)，最小フィールド幅(width)，精度(precision)， 長さ(length)， 変換指定子(type)それぞれを順に組み合わせた構文で指定する。 これにより詳細な指定で出力できる。

[かっこ] で括られた部分はオプションであり，必要な場合にのみ書けばよい。 [ ] はソースコードに含める文字ではなく領域を表している。

フラグ

フラグ説明
-左寄せで出力（デフォルトは右寄せ）
+必ず符号付きで出力（デフォルトはマイナスのみ）
半角スペース正の符号の場合，その位置に半角スペースを出力 （デフォルトは何も出力されず）
0出力する文字数が最小フィールド幅未満の場合には，それまで 0 を出力する
' (アポストロフィ)10 進数の場合，1000 単位でセパレータが表示される。

最小フィールド幅

最小フィールド幅は，出力する文字数の最小を指定する。 つまり，出力する文字数が指定したものより大きい場合には削られることなく表示される。

精度

精度は，出力の最大文字数を指定するものである。 浮動小数点数を表すデータ型の場合には，小数点以下の桁数を指定する。 文字列を表すデータ型の場合には，先頭から表示する最大の文字数を指定する。 （そこから末端まで切り捨てられ表示されない。）
